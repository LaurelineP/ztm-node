Implementation

Accessing this page triggers a loop where Node cannot delegates to other process as: - it is not a request that v8 should handle - it is not some file to process with libuv This is the main process handling the pure javascript.

The experience was to open twice this very page: then refresh both tab as quick as possible --> we can observe that their first state are pending then the first refreshed tab is done finally it is only when the first tab refreshed is done that the server process the second process --> == pended while first task has been processed / kept pending because of the loop effect 

	const express = require('express');
	const app = express();

	/**
	* Delay using a while loop with a "startTime"
	* ( avoiding to use process handling timers ( from v8 ))
	* @param {number} duration 
	*/
	function delay( duration ){
		const startTime = new Date();

		while( new Date() - startTime < duration  ){
			count += 1;
			console.log(`Within loop...: ${ count }`);
			
			/** Event loop & CPU is blocked processing
			* 	this loop
			* **/
		}
	}

	app.get( '/blocking-server', (req, res) => {
		delay(9_000);
		res.sendFile( pathResolve( '../pages/blocking-server.html' ) );
